---
title: 'Permissions Reference'
description: 'Complete list of all permissions and what they control.'
---

Decoda uses role-based access control to manage what each staff member can see and do. Every user is assigned a role, and each role has a set of permissions drawn from the modules listed below. Admins automatically have all permissions.

When you change a role's permissions, the update applies immediately to every user with that role. For instructions on creating and editing roles, see [Users, Roles & Permissions](/modules/settings/users).

## Access

| Permission | What It Does |
|---|---|

## Clock In

| Permission | What It Does |
|---|---|
| Export Clock-In History | Export clock-in history data to CSV/Excel |
| Edit Clock In/Out Times | Edit clock in and clock out times for all users |
| Skip Clock In | User is exempt from clocking in or out. Clock-in prompts and reminders are hidden for users with this permission. |
| View Clock In History | View clock in and clock out history for all users |
