Every change to a care plan is recorded automatically. You can view the full edit history at any time to see exactly what happened, who made the change, and when it occurred. This is useful for reviewing past decisions, understanding how a plan evolved, and keeping your team accountable.

## Opening the Edit History

<Steps>
  <Step title="Go to the Patient's Overview Tab">
    Navigate to the patient's chart and open the **Overview** tab.
  </Step>
  <Step title="Find the Care Plan">
    Locate the care plan you want to review in the **Care Plans** section.
  </Step>
  <Step title="Click Show History">
    Click the **Show history** link on the care plan card. A window opens showing the full edit history.
  </Step>
</Steps>

<Info>
  The **Show history** link appears on care plan cards on the patient's Overview tab. It is not available in the small card view (e.g., in the patient sidebar).
</Info>

## Reading the Edit History

The history window has two panels:

- **Left panel — Timeline of changes.** Each entry represents one change (or a group of related changes). The most recent change appears at the top. Each entry shows:
  - An icon indicating the type of change (created, edited, status change, or removed).
  - A short summary of what happened.
  - Who made the change and when.
  - For grouped changes, the number of individual changes included (e.g., "5 changes").

- **Right panel — Snapshot of the care plan.** Shows what the care plan looked like at the selected point in time. Steps that were affected by the selected change are highlighted so you can quickly spot what changed.

Click any entry in the left panel to see the care plan as it looked at that point.

## What Gets Tracked

The system records changes automatically — you do not need to turn anything on. The following types of changes are tracked:

### Care Plan Changes

| Change | Example |
|--------|---------|
| **Created** | A new care plan was assigned to the patient |
| **Name changed** | "Post-Surgery Recovery" → "Post-Op Recovery Plan" |
| **Description changed** | Updated the plan description |
| **Start date changed** | Moved the start date from March 1 to March 15 |
| **Tags changed** | Added "Physical Therapy" tag, removed "General" tag |
| **Location changed** | Moved from Main Office to Downtown Clinic |
| **Assigned by changed** | Changed the assigning provider |
| **Deleted** | The care plan was removed from the patient's record |

### Step Changes

| Change | Example |
|--------|---------|
| **Step added** | A new appointment or task step was added to the plan |
| **Step removed** | A step was deleted from the plan |
| **Status changed** | Step moved from Pending → Scheduled → Completed |
| **Services changed** | Added "Botox Treatment" to the appointment step |
| **Medications changed** | Removed "Ibuprofen" from the step |
| **Measurements changed** | Added Blood Pressure to the vitals for this visit |
| **Scheduled date changed** | Moved the step from April 1 to April 8 |
| **Step description changed** | Updated the provider notes on a step |
| **Repeat settings changed** | Changed from 3 repetitions to 4, or adjusted the interval |
| **Task fields changed** | Updated the task title, priority, assignee, or due date |

### Automatic Changes

Some changes happen automatically when linked appointments or tasks are updated. These are clearly labeled so you can tell them apart from manual edits:

| Source | What it means |
|--------|---------------|
| **User** | A staff member made this change manually |
| **Appointment sync** | The step status was updated automatically because the linked appointment changed (e.g., appointment completed → step marked Completed) |
| **Task sync** | The step status was updated automatically because the linked task changed (e.g., task cancelled → step marked Skipped) |
| **Repeat iteration** | The system automatically created the next repetition of a repeating step |

## How Changes Are Grouped

When several changes happen at the same time — for example, when you create a care plan with multiple steps, or save edits to several fields at once — they appear as a single entry in the timeline rather than as separate entries. The entry shows the total number of individual changes (e.g., "8 changes") so you know how much was included.

Click a grouped entry to see all the changes reflected in the snapshot on the right.

## Understanding the Snapshot View

When you select a change in the timeline, the right panel shows the care plan as it looked at that moment. Changes from the selected entry are visually highlighted:

| Highlight | Meaning |
|-----------|---------|
| **Added** (green) | This step was newly added in this change |
| **Removed** (red) | This step was removed in this change |
| **Modified** (blue) | One or more fields on this step were changed |
| **Unchanged** | This step was not affected by this change |

Click on a highlighted step to see the specific fields that changed, including the old and new values.

## Common Questions

**Is edit history available for templates?**
No. Edit history is only tracked for care plans assigned to patients, not for templates.

**Can I undo a change from the history?**
No. The edit history is read-only — it shows what happened but does not allow you to revert changes. To undo a change, edit the care plan directly.

**How far back does the history go?**
The history includes every change since the care plan was first created. Nothing is deleted or expired.

**Who can view edit history?**
Anyone with access to the patient's care plans can view the edit history.
