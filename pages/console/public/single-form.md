# Single Form Page - User Actions Guide

## Overview

The Single Form page allows patients to complete a single consent or intake form. It's accessed via a unique form path and requires phone verification before submission.

---

## Available Actions

### 1. Verify Identity via MFA

**Purpose:** Confirm your identity before accessing the form.

**Step-by-Step Instructions:**

1. **View the verification prompt**
   - MFA dialog appears automatically
   - Practice logo and name displayed

2. **Enter your phone number**
   - Type your 10-digit mobile number
   - Format: (555) 123-4567

3. **Request verification code**
   - Click "Send Code" or "Continue"
   - SMS with 6-digit code is sent

4. **Enter the verification code**
   - Type the code from SMS
   - Code expires after a few minutes

5. **Complete verification**
   - Click "Verify"
   - Form becomes accessible

---

### 2. Complete Form Fields

**Purpose:** Fill out all required and optional form fields.

**Step-by-Step Instructions:**

1. **Read form title and instructions**
   - Located at the top
   - Explains what information is collected

2. **Fill in form fields:**

   **Text Fields:**
   - Click in field
   - Type your response

   **Date Fields:**
   - Click calendar icon
   - Select date from picker

   **Dropdown/Select:**
   - Click to open options
   - Select your choice

   **Radio Buttons:**
   - Click one option
   - Only one can be selected

   **Checkboxes:**
   - Click to check/uncheck
   - Multiple can be selected

   **File Upload:**
   - Click "Choose File"
   - Select file from device
   - Wait for upload

   **Signature:**
   - Draw signature with mouse/finger
   - Click "Clear" to restart
   - Click "Accept" when done

3. **Handle conditional fields**
   - Some fields appear based on previous answers
   - Complete required fields first

---

### 3. Navigate Form Sections

**Purpose:** Move through different sections of the form.

**Step-by-Step Instructions:**

1. **Scroll through sections**
   - Use mouse wheel or swipe
   - Form sections displayed vertically

2. **Use section navigation (if available)**
   - Click section tabs if present
   - Jump to specific sections

3. **Required field indicators**
   - Asterisk (*) marks required fields
   - Must complete before submission

---

### 4. Submit the Form

**Purpose:** Send your completed form to the practice.

**Step-by-Step Instructions:**

1. **Ensure all required fields are complete**
   - Required fields marked with *
   - Invalid fields highlighted

2. **Review your answers**
   - Scroll through to verify
   - Make corrections as needed

3. **Click Submit button**
   - Located at bottom of form
   - Button may say "Submit", "Complete", etc.

4. **Wait for confirmation**
   - Loading indicator appears
   - Success message on completion

5. **View confirmation**
   - "Form Submitted Successfully" or similar
   - Thank you message displayed

---

### 5. Handle Form Not Found

**Purpose:** Respond to errors when form doesn't exist.

**When This Appears:**
- Form path is invalid
- Form has been deleted
- Form is not active

**What You'll See:**
- "Form Not Found" message
- Instructions to contact practice

**What to Do:**
- Verify the URL is correct
- Contact the practice for a new link
- Check if form link has expired

## Form Field Types

| Type | Description | How to Complete |
|------|-------------|-----------------|
| Text | Short text input | Type response |
| Textarea | Long text input | Type detailed response |
| Number | Numeric input | Enter numbers only |
| Email | Email address | Enter valid email |
| Phone | Phone number | Enter phone number |
| Date | Date selection | Pick from calendar |
| Time | Time selection | Select time |
| Select | Single choice dropdown | Choose one option |
| Multi-select | Multiple choice | Choose multiple options |
| Radio | Single choice buttons | Click one option |
| Checkbox | Multiple choice boxes | Check multiple options |
| File | File upload | Upload document/image |
| Signature | Signature capture | Draw your signature |

---

## Conditional Logic

Some forms use conditional logic:
- Fields may appear/hide based on answers
- Complete visible fields first
- New fields may appear after answering

---

## Session Recording

- Session is recorded for quality improvement
- Recording starts when form is verified
- Helps practice improve form experience

---

## Mobile Considerations

- Forms are mobile-responsive
- Touch-optimized inputs
- Camera access for file uploads
- Signature works with finger/stylus

---

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Form not loading | Check internet, refresh page |
| Verification code not received | Check SMS, request new code |
| Can't submit form | Complete all required fields |
| File upload fails | Check file size/type limits |
| Signature not saving | Ensure you clicked Accept |
| Form not found | Verify URL, contact practice |
